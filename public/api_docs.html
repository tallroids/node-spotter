Spotter REST API Documentation

How to use this page

The following calls are available for public consumption. Requests should be routed through the application according to the following protocol: https://warm-stream-94625.herokuapp.com/api/yourApiCallHere.

Locations

/api/getLocations

Returns all public locations

/api/getLocationById/:id

Returns a location object for specified id

/api/getLocationsByCategoryId/:id

Returns all public locations that are categorized by specified id

/api/createLocation

Creates a location with the following parameters in the body of the request:

  • title (STRING)
  • description (STRING)
  • lat (NUM)
  • lon (NUM)
  • isPublic (BOOL)

/api/updateLocation

Updates a location with the following parameters in the body of the request:

  • id (INT)
  • title (STRING)
  • description (STRING)
  • isPublic (BOOL)

/api/deleteLocation/:id

Deletes a location with the specified id

Categories

/api/getCategories

Returns an array of category objects

/api/createCategory

Creates a category with the title in the body of the request

/api/removeCategory/:id

Removes the category with the specified id

Favorites

/api/getFavoritesForUser/:id

Returns an array of location objects that a user has saved by the specified userId. Use 'self' to retrieve a list for the current user.

/api/addFavorite

Associates a location as a favorite for a specified user by passing the following parameters in the body of the request:

  • userId (INT)
  • locationId (INT)

/api/removeFavorite/:id

Removes the favorite with the specified id

Locations-Categories

/api/addLocationCategory

Associates a location with a category by passing the following parameters in the body of the request:

  • locationId (INT)
  • categoryId (INT)
